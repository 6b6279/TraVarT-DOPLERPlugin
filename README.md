  <div class="stackedit__html"><h1 id="doplerplugin">DOPLERPlugin</h1>
<p>The DOPLERPlugin is a plugin for the <a href="https://github.com/SECPS/TraVarT">TraVarT</a> ecosystem.<br>
It enables the conversion of UVL feature models to Dopler decision models.</p>
<h2 id="getting-started">Getting Started</h2>
<p>The project is built with Maven and uses JDK 23.<br>
If you want to use or work on the project, simply clone it with git and import it as a maven project in the IDE of your choice.<br>
To check the installation, run <code>maven verify</code> in the project root. The installation is working, when all tests (~300) run and pass.</p>
<p>The entry point into the code is the class <code>DoplerPlugin</code>. From there you get access to the <code>Serialiser</code>, <code>Deserialiser</code>, <code>Transformer</code> and <code>PrettyPrinter</code></p>
<ul>
<li>The <code>Transformer</code> is responsible for transforming an UVL feature model to a Dopler decision model</li>
<li>The <code>Serialiser</code> is responsible for writing a Dopler decision model into a string</li>
<li>The <code>Deserialiser</code> is responsible for deserializing a string and creating a Dopler decision model with it</li>
<li>The <code>PrettyPrinter</code> is responsible for transforming a Dopler decision model into different representations like a table</li>
</ul>
<h2 id="transformations">Transformations</h2>
<p>The <code>Transformer</code> functionality is divided into two parts:</p>
<ol>
<li>A Dopler to UVL part that converts a Dopler decision model to an UVL feature model</li>
<li>An UVL to Dopler part that converts an UVL feature model to a Dopler decision model</li>
</ol>
<p>There are also two different strategies for the transformation:</p>
<ol>
<li>ONEWAY - With this strategy all information is translated from one model to another that makes sense in the target model (e.g. mandatory features in UVL are not translated because there is no decision to make)</li>
<li>ROUNDTRIP - With this strategy as much information as possible is kept (the target model will contain redundant and superfluous elements)</li>
</ol>
<p>In the following sections the two parts, including an outline of the transformation, are described.<br>
The Transformations use a rule-based approach.</p>
<h3 id="uvl-feature-model-to-dopler-decision-model">UVL feature model to Dopler decision model</h3>
<p>The transformation from UVL to Dopler is a two-step process:</p>
<ol>
<li>Decisions are created from the feature tree</li>
<li>Rules are created from the conditions and then distributed to the Decisions.</li>
</ol>
